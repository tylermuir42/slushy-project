# Learner Lab Deliverables

Use these files as the final package for the Slushie learner-lab case study:

- `CASE_STUDY.md`
  - main write-up
  - learner-lab architecture
  - findings, insights, and recommendations

- `SLIDE_OUTLINE.md`
  - simple presentation structure

- `COST_ESTIMATE.md`
  - low-cost learner-lab monthly estimate

- `TEAM_SPRINT.md`
  - 4-person role split and short sprint plan

- `ARCHITECTURE_PROMPT.md`
  - prompt for Lucidchart AI

- `sample_machine_summary.json`
  - sample machine output for the proof of concept

- `dashboard.html`
  - lightweight demo dashboard based on the sample output

- `analyze_sample.py`
  - script that generates the sample summary from the raw JSON

## Minimum Demo Flow
1. Show the architecture diagram
2. Explain that the JSON is simulated telemetry
3. Show `sample_machine_summary.json` or `dashboard.html`
4. Highlight the three main business questions:
   - Which machines are down?
   - Which machines are at risk?
   - Which machines are selling the most?
5. End with the cost estimate and recommendations
